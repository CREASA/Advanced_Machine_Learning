# 2 types of error
r2_lr_list = []
mse_lr_list = []

for idx_train, idx_test in cv_split.split(X_full, y_full):
    X_train = X_full.iloc[idx_train]
    X_test = X_full.iloc[idx_test]
    y_train = y_full.iloc[idx_train]
    y_test = y_full.iloc[idx_test]
    
    # linear regression
    m_lr = lr.fit(X_train, y_train)
    y_pred = m_lr.predict(X_test)
    
    # R^2
    r2_lr = r2_score(y_true=y_test, y_pred=y_pred)
    r2_lr_list.append(r2_lr)
    
    # mean squared error
    mse_lr = mean_squared_error(y_true=y_test, y_pred=y_pred)
    mse_lr_list.append(mse_lr)

print("average R2 score for linear regression:", np.mean(r2_lr_list))
print("average MSE linear regression:", np.mean(mse_lr_list))

We can observe that the linear regression model outperforms the KNN model on both metrics - it has a higher R2 score and a lower MSE. A higher R2 score indicates that the linear regression model explains more of the variance, and a lower MSE indicates that the predictions from the linear regression model are closer to the actual observations. So we can say the linear regression model has better performance.

Think Again: How do the results compare to the previous KNN model without scaling? Why?

Scaled KNN Results (with scaling):

R2 score: 0.9288492935982386
MSE: 69669.86742831541

Previous KNN Results (without scaling):

R2 score: 0.8844743932435826
MSE: 104455.40707885305

We can observe that the KNN model perform better after scaling. It has a lower MSE(69669.9 compared with unscaled 104455.4) and a higher R2 score(0.93 compared with unscaled 0.88). When utilizing the KNN algorithm, feature scaling is paramount. If we don't scale our data, a feature with a broader range might overshadow others, compromising model accuracy. By scaling, each feature is given equal influence, yielding better results. It's like creating a level playing field for all features, which is why the scaled KNN model outperformed the unscaled one.
